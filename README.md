# 🚀 Workflow GitHub Kelompok Algoritma

Halo semuanya 👋  
README ini dibuat supaya semua anggota kelompok bisa mengikuti alur pengerjaan project GitHub dengan mudah, terutama yang masih pertama kali memakai GitHub.

---

# 📌 Langkah Pengerjaan

## 1️⃣ Clone Repository

Clone repository dulu ke laptop masing-masing.

```bash
git clone LINK_REPOSITORY
```

Contoh:

```bash
git clone https://github.com/nama/project.git
```

---

## 2️⃣ Masuk ke Folder Project

```bash
cd nama-project
```

Contoh:

```bash
cd project-algoritma
```

---

## 3️⃣ Buka di VSCode

```bash
code .
```

Kalau command `code .` tidak bisa, buka folder project secara manual lewat VSCode.

---

# 🌱 Membuat Branch Sendiri

⚠️ Jangan langsung mengerjakan di branch `main`.

Buat branch masing-masing:

```bash
git checkout -b nama-branch
```

Contoh:

```bash
git checkout -b tugas-kimi-[apa tugasnya]
```

---

# 💻 Mengerjakan Tugas

- Kerjakan bagian masing-masing.
- Pastikan program tidak error.
- Simpan file sebelum commit.

---

# 📤 Upload Perubahan ke GitHub

## 1. Tambahkan semua perubahan

```bash
git add .
```

---

## 2. Commit perubahan

```bash
git commit -m "menambahkan fitur sorting"
```

Gunakan pesan commit yang jelas ya 👍
Gunakan format commit yang rapi dan jelas supaya semua anggota mudah memahami perubahan yang dilakukan.

---

# 📌 Format Umum

```bash
git commit -m "jenis: deskripsi"
```

Contoh:

```bash
git commit -m "feat: menambahkan algoritma sorting"
```

---

# 📂 Jenis Commit

| Jenis | Fungsi |
|--------|---------|
| `feat` | Menambahkan fitur baru |
| `fix` | Memperbaiki bug/error |
| `docs` | Update dokumentasi / README |
| `style` | Merapikan code tanpa mengubah logic |
| `refactor` | Mengubah struktur code |
| `test` | Menambahkan testing |
| `init` | Commit awal project |

---

# ✅ Contoh Commit yang Benar

## Menambahkan fitur

```bash
git commit -m "feat: menambahkan menu login"
```

```bash
git commit -m "feat: menambahkan algoritma bfs"
```

---

## Memperbaiki error

```bash
git commit -m "fix: memperbaiki bug input array"
```

```bash
git commit -m "fix: memperbaiki error segmentation fault"
```

---

## Update dokumentasi

```bash
git commit -m "docs: update README workflow"
```

---

## Merapikan code

```bash
git commit -m "style: merapikan indentation"
```


## 3. Push ke GitHub

```bash
git push origin nama-branch
```

Contoh:

```bash
git push origin fitur-sorting
```

---

# 🔄 Jika Ada Update Baru dari Teman

Sebelum mulai kerja lagi, update project dulu:

```bash
git pull origin main
```

---

# ✅ Setelah Selesai

Kalau tugas sudah selesai:
- kasih tahu ketua/teman
- lakukan review
- baru merge ke branch `main`

---

# ⚠️ Rules Kelompok

- Jangan push langsung ke `main`
- Gunakan branch masing-masing
- Jangan hapus file teman
- Commit sesering mungkin
- Kalau bingung/conflict → tanya dulu

---

# 🆘 Command Penting

| Fungsi | Command |
|--------|---------|
| Clone repo | `git clone LINK` |
| Masuk folder | `cd nama-folder` |
| Buat branch | `git checkout -b nama-branch` |
| Tambah perubahan | `git add .` |
| Commit | `git commit -m "pesan"` |
| Push | `git push origin nama-branch` |
| Update project | `git pull origin main` |

---

# 🎯 Goal

Tujuan workflow ini:
- supaya kerja kelompok lebih rapi
- menghindari file ketimpa
- mempermudah revisi
- belajar GitHub bareng 🚀
